POUR Principles — Optional · Things to look out for and consider
POUR is the accessibility framework underlying WCAG. These are not mandatory fixes but good practice considerations — awareness of these will help catch issues earlier and improve the overall quality of the dashboard for all users.
P
Perceivable
Information must be presentable to users in ways they can perceive
KPI amber numbers1.4.11
Warning-400 #FDB022 on white — 2.1:1 fails→ Warning-800 #93370D on white — 7.2:1 ✓
Selected chip fill1.4.3
Brand-400 #8A7AF8 — 2.8:1 fails 4.5:1→ Brand-800 #1A299A + Brand-25 text — 10.1:1 ✓
Donut — colour-only segments1.4.1
All segments colour-only — fails for colour-blind users→ Brand-700 ■ · Warn-700 — · Error-700 ○ + shapes
Filter inputs + labels1.4.3
White on mid-purple ~3.1:1 · labels ~3.4:1 fails→ White inputs, Brand-800 text — 10.1:1 ✓
Tooltip contrast1.4.3
#888 tooltip bg — white text 2.3:1 fails→ Gray-900 #101828 bg + Gray-25 text — 16:1 ✓
Chart + table label text1.4.3
#888 muted grey on white — 3.0:1 fails→ Gray-700 #30374F on white — 11.8:1 ✓
Charts — no text alternative1.1.1
SVG/canvas with no aria-label — screen readers get nothing→ role="img" aria-label="[data summary]" on every chart
O
Operable
UI components must be operable by all input methods
No focus indicator anywhere2.4.7
Every element has outline:none — keyboard navigation invisible→ :focus-visible { outline: 3px solid #2D32C7; offset: 3px; }
Filter panel — no focus ring2.4.7
Selects and Reset button unreachable by keyboard→ .filter-panel :focus-visible { outline: 3px solid #E9E1FE; }
Chart popup — mouse only2.1.1
Detail card only shows on mouse hover→ Trigger on :focus · role="dialog" · Escape closes
Chip group — no keyboard pattern2.1.1
No roving tabindex · Space/Enter not handled→ role="group" + role="checkbox" + Space = toggle
Focus not obscured (new 2.2)2.4.11
Focused element may be hidden behind sticky header on scroll→ Ensure sticky header does not cover focused elements
Touch targets too small2.5.5
Chips and selects below 44×44px minimum→ min-height: 44px · min-width: 44px on all interactive elements
U
Understandable
Content and operation must be understandable
Version label "1.4.1" cryptic4.1.2
Sighted users guess what it means — no accessible label→ aria-label="Dashboard version 1.4.1"
Filter changes — silent4.1.3
Selecting a filter silently updates charts — no announcement→ aria-live="polite": "Showing 17 results for Latest Month"
Inconsistent filter layout3.2.4
CSS screen has different filter slots to Qualitative screen→ Standardise order. Disable irrelevant filters with aria-disabled
Truncated labels2.4.6
"Committee Managem..." — full text not accessible without mouse→ Full label in aria-label on each bar + tooltip on focus
No heading hierarchy2.4.6
Title + section headings are styled text, not H1/H2/H3→ H1 = page title · H2 = chart sections · role="main"
R
Robust
Content must work with current and future assistive technologies
Chip group — no ARIA roles4.1.2
No role="group" · no aria-checked · announced as generic divs→ role="group" + role="checkbox" + aria-checked="true/false"
Data table — no scope1.3.1
No scope="col" — screen readers cannot associate headers→ scope="col" all th · aria-label on table element
Off-system colours in code1.4.3
#9B96D8 · #D85A30 · #888 not in Design System — fragile→ Replace with Brand-700 · Error-700 · Gray-700 tokens
SVG text not scalable1.4.4
Chart axis text below 12px · doesn't scale with browser zoom→ Min 12px all axis labels · test at 200% browser zoom
Radio group — no fieldset1.3.1
Assurance Period radio buttons lack grouping markup→ fieldset + legend · role="radiogroup" · Arrow keys navigate
